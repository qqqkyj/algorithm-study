# 📚 알고리즘 기초 정리

## ⏱ 시간 복잡도 (Time Complexity)

시간 복잡도는 **입력 크기 n이 증가할 때 알고리즘의 연산 횟수가 얼마나 증가하는지**를 나타내는 지표이다.  
일반적으로 **Big-O 표기법**을 사용하여 표현한다.

### Big-O 표기법

알고리즘의 **최악의 경우 시간 복잡도**를 표현하는 방식이다.

대표적인 시간 복잡도 증가 속도

| 시간 복잡도 | 의미      |
| ----------- | --------- |
| O(1)        | 상수 시간 |
| O(log n)    | 로그 시간 |
| O(n)        | 선형 시간 |
| O(n log n)  | 선형 로그 |
| O(n²)       | 이차 시간 |
| O(2ⁿ)       | 지수 시간 |

---

## 📌 시간 복잡도 예시 (1부터 n까지의 합)

### 1️⃣ 반복문 사용

```java
int sum = 0;
for(int i = 1; i <= n; i++){
    sum += i;
}
```

연산 횟수

```
1 + 2 + 3 + ... + n
```

반복문이 **n번 실행**되므로

```
O(n)
```

---

### 2️⃣ 수학 공식 사용

1부터 n까지의 합 공식

```
1 + 2 + ... + n = n(n+1)/2
```

```java
int sum = n * (n + 1) / 2;
```

연산이 **한 번만 수행**

```
O(1)
```

👉 같은 문제라도 **알고리즘에 따라 시간 복잡도가 달라질 수 있다.**

# 📦 자료 구조 기초

자료 구조는 **데이터를 효율적으로 저장하고 관리하기 위한 구조**이다.

알고리즘 문제에서 자주 사용하는 기본 자료구조

- 스택 (Stack)
- 큐 (Queue)
- 덱 (Deque)

---

# 📚 스택 (Stack)

## 특징

- **LIFO (Last In First Out)**
- **후입선출**

즉, **나중에 들어온 데이터가 먼저 나간다.**

## 구조

```
push(1)
push(2)
push(3)

3 ← top
2
1
```

```
pop()

2 ← top
1
```

## 주요 연산

| 연산       | 설명                   |
| ---------- | ---------------------- |
| push       | 데이터 삽입            |
| pop        | 데이터 제거            |
| peek / top | 가장 위 데이터 확인    |
| isEmpty    | 스택이 비어있는지 확인 |

## 시간 복잡도

| 연산 | 시간 복잡도 |
| ---- | ----------- |
| push | O(1)        |
| pop  | O(1)        |
| peek | O(1)        |

## 사용 예시

- 괄호 검사
- 후위 표기식 계산
- 재귀 호출
- DFS

---

# 📚 큐 (Queue)

## 특징

- **FIFO (First In First Out)**
- **선입선출**

즉, **먼저 들어온 데이터가 먼저 나간다.**

## 구조

```
offer(1)
offer(2)
offer(3)

1 2 3
↑     ↑
front rear
```

```
poll()

2 3
↑   ↑
```

## 주요 연산

| 연산            | 설명              |
| --------------- | ----------------- |
| offer / enqueue | 데이터 삽입       |
| poll / dequeue  | 데이터 제거       |
| peek            | 맨 앞 데이터 확인 |

## 시간 복잡도

| 연산    | 시간 복잡도 |
| ------- | ----------- |
| enqueue | O(1)        |
| dequeue | O(1)        |
| peek    | O(1)        |

## 사용 예시

- BFS (너비 우선 탐색)
- 프린터 대기열
- 작업 스케줄링

---

# 📚 덱 (Deque)

## 특징

- **Double Ended Queue**
- 양쪽에서 삽입과 삭제 가능

즉,

- **앞에서도 넣고 뺄 수 있음**
- **뒤에서도 넣고 뺄 수 있음**

## 구조

```
addFirst(1)
addLast(2)
addLast(3)

1 2 3
```

```
removeLast()

1 2
```

## 주요 연산

| 연산        | 설명        |
| ----------- | ----------- |
| addFirst    | 앞에 삽입   |
| addLast     | 뒤에 삽입   |
| removeFirst | 앞에서 삭제 |
| removeLast  | 뒤에서 삭제 |

## 시간 복잡도

| 연산        | 시간 복잡도 |
| ----------- | ----------- |
| addFirst    | O(1)        |
| addLast     | O(1)        |
| removeFirst | O(1)        |
| removeLast  | O(1)        |

## 사용 예시

- 슬라이딩 윈도우
- BFS 최적화
- 양방향 탐색

---
